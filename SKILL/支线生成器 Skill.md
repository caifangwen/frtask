# 支线生成器 Skill

## 核心任务

根据用户输入的 **关键词** 或 **核心句**，生成一份完整的支线任务文档。

---

## 输入解析规则

用户输入可能是以下形式之一：

| 输入形式 | 示例 |
|----------|------|
| 单个关键词 | `健身`、`读书`、`重构代码` |
| 关键词组合 | `产品调研 竞品分析` |
| 一句核心句 | `把用户反馈整理成需求文档` |
| 带优先级的描述 | `紧急：修复登录页面 bug` |

解析时提取：
1. **任务标题**：简洁动宾短语，≤15字
2. **目标描述**：一句话说清楚"做什么、达成什么"
3. **初始步骤**：拆解3-5个可执行的子任务
4. **优先级判断**：根据关键词推断（紧急/重要 → 高；日常 → 中；探索/学习 → 低）
5. **预计时长**：根据任务复杂度估算（简单7天、中等14天、复杂30天）

---

## 输出模板

严格按照以下格式输出，今天日期用实际日期填入：

```markdown

状态: 待启动
优先级: [1/2/3]
优先级标签: [高/中/低]

预计完成: [YYYY-MM-DD]
负责人: 自己
阻塞原因: 
标签: [[标签1], [标签2]]

- [ ] [步骤二：...]
- [ ] [步骤三：...]

- [ ] [步骤五：可选]

## 工作记录

- 初始化支线
```

---

## 优先级映射规则

- **优先级 1（高）**：含"紧急"、"urgent"、"deadline"、"修复"、"bug"、"上线"、"客户"
- **优先级 2（中）**：默认，日常任务、功能开发、调研
- **优先级 3（低）**：含"探索"、"学习"、"研究"、"尝试"、"了解"

---

## 标签推断规则

根据关键词自动推断1-3个标签：

- 涉及代码/技术 → `技术`
- 涉及文档/写作 → `写作`
- 涉及用户/产品 → `产品`
- 涉及学习/读书 → `学习`
- 涉及沟通/会议 → `沟通`
- 涉及设计/UI → `设计`
- 涉及数据/分析 → `数据`

---

## 步骤拆解原则

每个步骤需满足：
- **可执行**：动词开头，明确操作对象（如"整理X"、"调研Y"、"写Z文档"）
- **可验证**：完成后有明确产出
- **粒度适中**：单步骤预计1-4小时可完成

典型拆解框架（根据任务类型选择）：

**调研类**：收集资料 → 整理信息 → 提炼结论 → 输出文档  
**开发类**：明确需求 → 设计方案 → 编码实现 → 测试验证 → 上线部署  
**写作类**：确定大纲 → 收集素材 → 初稿撰写 → 修改润色 → 定稿发布  
**学习类**：选定资料 → 阅读/观看 → 整理笔记 → 实践应用 → 总结复盘  

---

## 输出要求

1. **直接输出 Markdown 文档**，无需前置说明
2. 如果用户输入模糊，**先生成一个合理版本**，再附一句："如需调整标题/步骤/优先级，告诉我～"
3. 文件名建议：在文档末尾附上 `> 💾 建议文件名：[任务标题].md`
4. 所有日期使用真实当天日期，预计完成 = 开始日期 + 估算天数
