# 多虚拟桌面 × Obsidian 任务管理工作流

> 用虚拟桌面做任务隔离，用 Obsidian 做信息枢纽，总台统筹全局，支线专注推进。

---

## 核心理念

每个虚拟桌面 = 一个独立的任务上下文（心流空间）  
每个桌面配一个单文件 Obsidian Note = 该任务的实时工作台  
总台桌面 = 全局调度中心，持有完整仓库，俯瞰所有支线

---

## 桌面结构

```
桌面 0 ── 总台（Hub）
           └── Obsidian 仓库（vault）
               ├── _INDEX.md          ← 总任务看板
               ├── 支线-A.md
               ├── 支线-B.md
               ├── 支线-C.md
               └── ...

桌面 1 ── 支线 A
           └── 支线-A.md（单文件，独立窗口或 Obsidian 单页模式）

桌面 2 ── 支线 B
           └── 支线-B.md

桌面 N ── 支线 N
           └── 支线-N.md
```

---

## 总台（桌面 0）

### 职责
- 持有 Obsidian 仓库的**完整视图**
- 维护 `_INDEX.md`：所有支线的状态一览
- 接收各支线的进展更新
- 决策：新建支线、合并支线、关闭支线

### `_INDEX.md` 推荐格式

```markdown
# 总任务看板
更新时间：{{date}}

## 进行中
- [[支线-A]] — 正在做 XXX，预计完成 XX/XX
- [[支线-B]] — 等待反馈

## 待启动
- [ ] 支线-D：调研 YYY

## 已完成
- [x] [[支线-C]] — 已归档 ✓

## 今日重点
> 优先推进支线-A，支线-B 阻塞中暂缓
```

---

## 支线桌面（桌面 1~N）

### 职责
- 专注单一任务，不切换上下文
- 维护属于本支线的**单文件工作台**
- 完成阶段后将关键进展同步回总台 `_INDEX.md`

### 支线文件推荐格式

```markdown
# 支线-A：[任务名称]

## 状态
- 阶段：进行中
- 优先级：高
- 负责人：—
- 开始日期：YYYY-MM-DD

## 目标
> 一句话说清楚这条支线要达成什么。

## 当前进展
- [x] 步骤一
- [x] 步骤二
- [ ] 步骤三 ← 当前位置

## 工作记录
### YYYY-MM-DD
- 做了什么
- 遇到什么问题
- 下一步

## 待解决 / 阻塞
- 问题 X：等待 XX 回复

## 关联资源
- [[总台 _INDEX]]
- 参考链接、文件路径等
```

## 日常操作流程

### 开始新的工作日

```
1. 打开总台（桌面 0）
2. 浏览 _INDEX.md，确认各支线状态
3. 决定今日重点，在 _INDEX.md 写下「今日重点」
4. 切换到对应支线桌面，开始工作
```

### 切换支线

```
1. 在当前支线文件末尾记录「中断点」（下一步要做什么）
2. 切换到目标桌面
3. 读取该支线文件的「当前进展」，快速恢复上下文
```

### 支线有重要进展

```
1. 在支线文件「工作记录」里记录
2. 切换到总台桌面
3. 更新 _INDEX.md 对应支线的状态行
4. 切换回支线继续
```

### 结束工作

```
1. 在当前支线文件写好「下一步」
2. 切换到总台
3. 更新 _INDEX.md 各支线状态
4. （可选）写今日总结，标注明日优先项
```

## Obsidian 配置建议

### 仓库结构

```
vault/
├── _INDEX.md              # 总看板（置顶）
├── 支线/                  # 所有支线文件
│   ├── 支线-A.md
│   └── 支线-B.md
├── 归档/                  # 已完成支线移入此处
└── 素材/                  # 参考资料、草稿
```

### 推荐插件

| 插件 | 用途 |
|------|------|
| Dataview | 自动聚合各支线状态，_INDEX 可用查询替代手写 |
| Templater | 新建支线时一键生成模板 |
| Workspaces | 保存各桌面的 Obsidian 窗口布局 |
| Quick Switcher | 总台快速跳转到任意支线文件 |

### Dataview 自动看板（可选替代手写 _INDEX）

````markdown
```dataview
TABLE 状态, 优先级, 开始日期
FROM "支线"
WHERE 状态 != "已完成"
SORT 优先级 DESC
```
````

## 工作流优势

- **认知隔离**：每个桌面只看当前任务，不被其他支线干扰
- **快速恢复**：切换桌面即切换任务，单文件工作台让上下文恢复秒级完成
- **全局掌控**：总台一眼看清所有支线状态，不会遗漏
- **轻量归档**：完成的支线移入 `归档/` 文件夹，历史可查

## 快速参考卡

| 场景 | 操作 |
|------|------|
| 查全局状态 | 切总台 → 看 `_INDEX.md` |
| 新建支线 | 总台新建文件 → 用模板初始化 → 分配新桌面 |
| 支线阻塞 | 在支线文件标注阻塞原因 → 总台更新状态为「等待」 |
| 支线完成 | 支线文件做完结记录 → 总台标为完成 → 移入归档 |
| 找不到思路 | 回总台看 _INDEX，重新梳理优先级 |
